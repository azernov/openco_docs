Ниже приведена типовая структура проекта OpenColour.

Приведены только те каталоги и файлы, которые используются непосредственно при разработке.

```
db
|-dbdata                    -набор файлов с данными таблиц
|-dbstructure               -набор файлов со структурой таблиц
|-dbsample                  -тестовый набор данных для инициализации проекта (склейка всех файлов из папок dbdata и dbstructure)
|-db_backup.php             -скрипт автоматического создания файлов в трех вышестоящих папках
|-db_backup.config.inc.php  -конфигурационный файл для скрипта db_backup.php
|
source                      -Папка с проектом верстки. ВСЕ изменения стилей, скриптов и служебных изображений производятся ТОЛЬКО ЗДЕСЬ!
|-assets
|   |-fonts                 -Шрифты
|   |-i                     -Все служебные изображения храним здесь и изменяем тоже здесь!!!
|   |-jade                  -Этот каталог чаще всего игнорируем
|   |-js                    -Если нужно поменять js, меняем их здесь при запущенном gulp-таске "watch"
|   |-sass                  -Если нужно поменять стили, меняем исходные .scss-файлы при запущенном gulp-таске "watch"
|
|-gulpfile.js               -Gulp для верстальщиков (все таски этого файла нацелены только на текущий каталог source)
|-gulpfile_production.js    -Gulp для разработчика сайта (В 99% случаев используем его)
|
www
|-_development
|-assets                    -В этом каталоге ничего не меняем вручную, кроме каталога components
|   |-components
|   ...
|
....
|-core
|   ...
|   |-components
|   |   ...
|   |   |-gitmodx
|   |   |   |-config        -Каталог с актуальными конфиг-файлами
|   |   |   |-config.sample -Каталог с конфигами-примерами
|   |   |   |-elements      -Каталог с чанками/сниппетами/плагинами/шаблонами
|   |   |   ...
```
